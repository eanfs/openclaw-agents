# AGENTS.md - 工作区规范

## 工作区概述

本工作区专注于自动轮播内容生成和发布，包括网站分析、视觉设计、多平台发布和数据驱动优化。

## 核心职责

### 每日轮播管道
- 使用 Playwright 研究任何网站 URL，每天生成 6 张视觉连贯的幻灯片
- 使用 Gemini 的图像到图像功能生成幻灯片，第 1 张建立视觉 DNA，第 2-6 张参考它
- 通过 Upload-Post API 直接发布到 TikTok 和 Instagram——每天一条

### 视觉一致性引擎
- 使用 Gemini 的 image-to-image 功能，幻灯片 1 建立视觉 DNA，幻灯片 2-6 参考它
- 保持颜色、排版和美学的连贯性

### 分析反馈循环
- 通过 Upload-Post 分析端点获取性能数据
- 识别哪些钩子和风格有效
- 自动将洞察应用到下一个轮播

### 自我改进系统
- 在 learnings.json 中累积所有帖子的学习——最佳钩子、最佳时间、成功的视觉风格
- 让轮播 #30 大大优于轮播 #1

## 工作流程

### 阶段 1：从历史学习
1. **获取分析数据**：调用 Upload-Post 分析端点，获取配置文件指标和每个帖子表现
2. **提取洞察**：运行 learn-from-analytics.js，识别表现最佳的钩子、最佳发布时间和参与模式
3. **更新学习**：将洞察累积到 learnings.json 持久知识库
4. **规划下一个轮播**：读取 learnings.json，从表现最佳者中选择钩子风格，在最佳时间安排，应用建议

### 阶段 2：研究与分析
1. **网站抓取**：对目标 URL 运行 analyze-web.js，进行完整的基于 Playwright 的分析
2. **品牌提取**：用于视觉一致性的颜色、字体、logo、favicon
3. **内容挖掘**：从所有内部页面提取功能、推荐、统计、定价、行动号召
4. **细分检测**：分类业务类型并生成细分适当的叙事
5. **竞争者映射**：识别网站内容中提到的竞争者

### 阶段 3：生成与验证
1. **幻灯片生成**：运行 generate-slides.sh，调用 generate_image.py 通过 uv 创建 6 张幻灯片，使用 Gemini
2. **视觉一致性**：幻灯片 1 来自文本提示；幻灯片 2-6 使用 Gemini image-to-image，以 slide-1.jpg 作为输入参考
3. **视觉验证**：Agent 使用自己的视觉模型检查每张幻灯片的文字清晰度、拼写、质量，以及底部 20% 没有文字
4. **自动重生成**：如果任何幻灯片失败，只重新生成该幻灯片（使用 slide-1.jpg 作为参考图像），重新验证直到全部 6 张通过

### 阶段 4：发布与追踪
1. **多平台发布**：运行 publish-carousel.sh，推送 6 张幻灯片到 Upload-Post API
2. **热门音乐**：auto_add_music=true 为 TikTok 添加热门音乐，提升算法
3. **元数据捕获**：保存 API 响应中的 request_id，用于分析追踪
4. **用户通知**：在一切成功后报告发布的 TikTok + Instagram URL
5. **自动安排**：读取 learnings.json bestTimes，设置下次 cron 执行在最佳小时

## 成功标准

- 发布一致性：每天 1 条轮播，完全自主
- 观看量增长：每个轮播平均观看量环比增长 20%+
- 互动率：5%+（点赞 + 评论 + 分享 / 观看量）
- 钩子胜率：在 10 个帖子内识别前 3 个钩子风格
- 视觉质量：90%+ 的幻灯片在第一次 Gemini 生成时通过视觉验证
- 最优时间：发布时间在 2 周内收敛到最佳小时
- 学习速度：每 5 个轮播可衡量表现提升
- 跨平台覆盖：TikTok + Instagram 同时发布，平台特定优化
